BDP-DSN为 BDP 平台的统一数据存储服务。DSN SDK 提供了 BDP-DSN 服务的 Java API。使用 SDK，可以构建配合的使用的 Java 应用程序。

DSN-DB 提供了 DSN 服务的结构化数据存储读取的接口，通过该工具包，可以实现对 DSN 结构化数据的操作。

2、重新构建项目

配置 pom.xml 文件后，使用 Maven 构建项目。

2.使用SDK

结构化数据查询类操作：

1、创建客户端 DsnDb，配置 DSN 服务的地址和端口。

2、执行查询类操作，参数为目标的 SQL 语句。

结构化数据执行类操作：

1、创建客户端 DsnDb，配置 DSN 服务的地址和端口。

2、执行执行类操作，参数为目标的 SQL 语句。

二、DSN-Medata模块

DSN-Metadata 提供了 DSN 服务的元数据相关操作的接口，通过该工具包，可以实现对 DSN 元数据的管理。

1.设置Maven项目

1、在Maven项目的pom.xml文件中，添加开发工具包依赖。

2、重新构建项目

配置 pom.xml 文件后，使用 Maven 构建项目。

2.使用SDK

查看元数据列表：

1、创建客户端 DsnMetadata，配置 DSN 服务的地址和端口。

2、执行查看元数据列表。

删除文件：

1、创建客户端 DsnMetadata，配置 DSN 服务的地址和端口。

2、删除指定 id 的文件。

导出文件：

1、创建客户端 DsnMetadata，配置 DSN 服务的地址和端口。

2、配置MetadataExport，设置元数据 id ，导出目录，是否导出至本地等。

导入文件：

1、创建客户端 DsnMetadata，配置 DSN 服务的地址和端口。

研发管理流程规范(宇动源-BD-AI) V1.5

2022-09-13

一、目的及适用范围

1、目的

规范研发流程，提高协作效率。
专注研发工作，科学规范考核。

2、核心价值与原则

  • 抵制无计划、需求堆砌无目的式研发
  • 小型的、多元化的项目组（角色依赖和职责共享）
  • 以产品发布为中心，明确一致的目标
  • 分享产品的前景，所有人都参与设计
  • 认真从过去的项目中吸取经验
  • 共同管理，共同决策
  • 项目组成员在同一地点办公
  • 大型项目组也像小项目组一样运转

    3、适用范围

    本标准适用于产业运营中心的产品研发流程（项目开发、交付、售前支持仅作参考）。
    本标准的使用者必须是本企业（宇动源（北京）信息技术有限公司）内部的测试人员，开发人员。

    4、总体说明

    将研发流程拆分为需求分析，开发，测试，版本发布，演示 5个阶段
    每个阶段包含步骤，结果，规范，操作4个维度
    步骤: 该阶段需要完成的宏观工作。
    结果: 该阶段输出的交付产物。
    规范: 该阶段涉及到的具体规范。
    操作: 该阶段涉及到的工具使用。

二、 需求分析（里程碑的诞生）

1、步骤

  1. 由PO和产品经理共同宣讲（PO主讲，产品经理补充）截止评估前的产品原型和功能定义

  2. 技术团队根据全量的功能/需求，进行「任务地图」的评审和规划

    任务地图：一份截止评估前的产品全量功能/需求的罗列，及对应的每个任务的具体所需工时。

  3. 由PO来进行里程碑（任务地图的子集）划分，及团队对于里程碑任务的确定

    PO（Project Owner）：团队的记录者、会议纪要责任人、团队技术开发的润滑剂，通常由后端为主的角色担任。
    主要职责：

    • 完全理解产品定义、产品价值、产品业务逻辑
    • 与产品经理交涉技术的可行性（可行性低的需求，有否决权）
    • 站会的组织者，开发环节中里程碑的主要责任人（制定、评审、初验）
    • 产品与团队之间业务逻辑的最终解释人
    • 任务地图的最终解释人

  4. 材料上传到团队每一个人都能看到的位置
